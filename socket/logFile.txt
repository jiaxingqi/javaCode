	1、什么是Java的动态代理对象？
		（*）动态代理：是一种包装设计模式，在不修改源码的情况下，去增强（重写）某个类的方法
		（*）利用：Java API:  Proxy(java.lang.reflect.Proxy)
		（*）Java动态代理的案例（应用场景）：数据库连接池
			1、启动mysql数据库   service mysqld start
			2、配置：利用Hive配置
			         root用户登录   mysql -uroot
						MySQL数据库的配置：
						创建一个新的数据库：create database hive;
						创建一个新的用户：
						create user 'hiveowner'@'%' identified by 'password';
						给该用户授权
						grant all on hive.* TO 'hiveowner'@'%'; 
						grant all on hive.* TO 'hiveowner'@'localhost' identified by 'password';

五、HDFS的高级功能
	1、回收站
		（*）补充：Oracle数据库的回收站
		     删除表：drop table emp;  ------> 默认删除到：Oracle的回收站
			 
			 查看回收站：show recyclebin;
			 可以通过表在回收站中的名字去访问原来的表：
			       select * from "BIN$WASzmd+19oXgUAB/AQB2wg==$0";
		
			 清空回收站： purge recyclebin;
			 彻底删除一张表：drop table emp purge;
			 
			 通过Oracle的数据库的闪回（flashback）技术，进行恢复；11g：7种     10g：6种
			 1、闪回表
			 2、闪回查询
			 3、闪回删除: flashback drop 操作回收站
			 4、闪回版本查询
			 5、闪回事务查询
			 6、闪回数据库
			 7、闪回数据归档
			 
			 恢复回收站中的表：
			 flashback table emp to before drop;
		
			 如果闪回同名的表：
			 1、先恢复后删除的表
			 2、flashback table emp to before drop rename to emp_old;
		
		（*）HDFS的回收站：本质是：将删除的数据移动到一个隐藏目录下
			1、默认：禁用的
				日志：
				17/08/31 11:09:53 INFO fs.TrashPolicyDefault: Namenode trash configuration: Deletion interval = 0 minutes, Emptier interval = 0 minutes.
				Deleted /output
				
			2、启用回收站：core-site.xml中
					<property>
					  <name>fs.trash.interval</name>
					  <value>1440</value>
					</property>
					
	            日志：
				Moved: 'hdfs://192.168.157.111:9000/folder1' to trash at: hdfs://192.168.157.111:9000/user/root/.Trash/Current
